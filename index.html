The below modules reflect a step by step build up of Restaurantly, a simple restaurant review app that is test driven including capybara integration tests, uses haml, git branching, rails 4, ruby 2, rspec, devise, omniauth, twitter login, zurb-foundation and thoughtbot's factory-girl. The idea is that you watch the videos and then build your own clone of the app, learning as you go. The source code is cloneable at github.

If you are super new to coding and really want to learn how all this works, find me on koding.com/jcdavison and we can remote pair program a little to get you off and running.

About

Join me at rubyonrailstutor.com for a fun highly social, distributed real time pair programming class series where we will explore the fundamantels of web application development. We will master the fundamentals of programming tools like ruby, rails, git, haml, heroku, responsive web design, test driven development, vim, sublime, github and the intricacies of software product development.

Prior to 2012, I didn't know how to build software. Thanks to Dev Bootcamp and a ton of hard work and support from the San Francisco developer community, in the span of 3 rigorous training months and 18 months working, I now operate as a wholly independent software product tactition. I hope to offer you some tools to make the same transformation.
